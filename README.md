# Whisper → LJSpeech dataset (Piper Trainer)

Repositorio local preparado para convertir transcripciones de Whisper a un dataset estilo LJSpeech
apto para entrenamiento con Piper.

Requisitos principales
- Python 3.10+ (recomendada instalación oficial desde python.org)
- ffmpeg en PATH (necesario para decodificar/extraer audio)
- virtualenv recomendado

Instalación rápida

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install -r requirements.txt
```

Nota: `torch` puede requerir un comando específico según tu plataforma/GPU. Consulta https://pytorch.org.

Archivos principales y uso

- `youtube_audio_downloader.py`
  - Descarga el audio de un vídeo de YouTube con `yt-dlp` y extrae el audio (usa ffmpeg internamente).
  - Ejemplo:
    ```powershell
    python youtube_audio_downloader.py "https://youtu.be/XXXXXXXX" -o downloads --format mp3
    ```

- `whisper_to_json.py`
  - Transcribe un archivo de audio usando Whisper y guarda un JSON con `segments` y `phrases`.
  - Opciones importantes:
    - `-m` o `--model`: modelo Whisper (por defecto `large-v3`).
    - `-l` o `--language`: forzar idioma (ej. `es`).
    - `--normalize-phrases`: por defecto `True`, asegura que las `phrases` queden entre 1.5s y 7s.
  - Ejemplo:
    ```powershell
    python whisper_to_json.py downloads\video.mp3 -o salida.json -m large-v3 -l es
    ```

- `build_ljspeech_dataset.py`
  - Convierte el JSON generado por `whisper_to_json.py` en un dataset LJSpeech-style.
  - Crea la carpeta `dataset/wavs/` y `dataset/metadata.csv`.
  - Opciones importantes:
    - `--sample-rate` (`22050` por defecto, también `16000` soportado)
    - `--metadata-format` (`classic` o `piper`)
    - `--min-duration` (por defecto `0.4`s)
    - `--end-padding` (padding añadido al final del último word para evitar cortar sílabas; por defecto `0.25`s)
    - `--trim-silence` (intención para recortar silencio, no recorta palabras)
  - Ejemplo:
    ```powershell
    python build_ljspeech_dataset.py --input-json salida.json --output-dir .\dataset --sample-rate 22050 --metadata-format classic --end-padding 0.30
    ```

Notas sobre calidad del dataset
- Los archivos WAV exportados son mono, 16-bit PCM y remuestreados a la tasa solicitada mediante `ffmpeg`.
- El script usa marcas de tiempo de palabra (si están) y añade un pequeño `end-padding` para evitar cortar la última sílaba.
- Si necesitas dividir o normalizar más las frases, ajusta `--end-padding` o regenera con `--normalize-phrases` activo antes de construir el dataset.
